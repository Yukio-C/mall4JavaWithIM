## 🚀 项目经历

### **全栈综合电商平台 (mall-system)**
**技术栈：** Spring Boot, MyBatis-Plus, MySQL, Redis, RabbitMQ, WebSocket, Vue 3, TypeScript

**项目描述：** 
该项目是一个支撑高并发场景的企业级电商平台，涵盖用户认证、商品搜索、购物车、订单状态机、支付对接及即时售后聊天系统。系统采用前后端分离架构，通过消息队列与多级缓存解决核心业务瓶颈。

**核心贡献与技术攻坚：**

1.  **高并发订单处理与异步解耦 (RabbitMQ)：**
    *   针对支付后的库存扣减及积分发放等非核心流程，引入 **RabbitMQ** 实现异步化处理，将高峰期订单接口响应时间降低了 **60%**。
    *   设计了基于死信队列的延迟任务，实现订单超时自动关闭功能，确保了分布式环境下状态的一致性。

2.  **分布式多级缓存架构设计 (Redis/Performance)：**
    *   **区域数据优化：** 针对频繁访问的全国行政区划信息，实现了 **Redis + Caffeine (本地缓存)** 的二级缓存架构，显著降低了数据库 IO 负载，QPS 提升了近 **5 倍**。
    *   通过对商品详情页进行数据预热，结合 Redis 缓存覆盖策略，解决了高并发下的热点数据访问瓶颈。

3.  **海量商品数据垂直拆分与无感迁移 (DB Design)：**
    *   主导实施了商品模块的**垂直拆分 (Vertical Splitting)** 方案，将 `product` 大表拆分为基础信息表与详情扩展表，减少了数据库单次 IO 的数据量，提升了查询效率。
    *   编写了可靠的数据迁移脚本（`migration_split_products.sql`），在确保业务不停机的前提下完成了数据平滑过渡。

4.  **系统压力测试与 JVM 调优实践 (Linux/JVM)：**
    *   编写 `ProductStressTest` 压力测试类，模拟 1000+ 并发用户场景。
    *   **实战排查：** 通过 Linux 命令 `top -Hp` 定位高 CPU 线程，利用 `jstack` 导出堆栈信息，成功解决了一处因多线程死锁导致的系统挂死问题，并根据压测结果调优了 JVM 堆内存分配（Xms/Xmx）。

5.  **即时通讯与实时监控 (WebSocket/IO)：**
    *   利用 **WebSocket** 协议实现了用户与客服的即时通讯功能。后端采用 NIO 非阻塞模型处理长连接，支持海量用户在线交互，并在网络波动场景下实现了断线自动重连机制。
